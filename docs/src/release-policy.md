# Release policy

Releases follow the roadmap (one RFC set per minor version) and ship as
`dioxus-swdir-tree-<version>.tar.gz` archives. When a release ships, its RFCs
move from `rfcs/proposed/` to `rfcs/done/` with their Status updated.

**v1.0.0 is gated:** it is published only with explicit confirmation from the
project owner. No automation, schedule, or parity milestone overrides this.
